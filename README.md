# Adaptive Copolymer Self-Assembly — Research Code

This repository contains the code used to generate the results and figures for the paper:

Leveraging inhomogeneous binding of incompatible constituents for adaptive copolymer self-assembly, Soft Matter, 2025,21, 9018-9024 

The code implements an RPA analytical framework for computing polymer binding states, evaluating vertex functions, and performing large-scale parameter sweeps to generate phase diagrams.

## Repository structure

### vertex_calc/

Core Python modules implementing the analytical theory.

This folder contains reusable modules that:

- Define key parameters and gather into relevant system classes
- Compute polymer binding state statistics
- Evaluate two-, three-, and higher-order vertex functions

These modules are designed to be imported by both local workflows and large-scale parameter sweep scripts.

Purpose:
Core library layer of the project.

---

### data_gen_sherlock/

Scripts for large-scale phase diagram generation on high-performance computing (HPC) systems.

This folder includes:

- A copy of the vertex_calc modules (for portability in cluster jobs)
- Four parallelized driver scripts beginning with:

  multipros_phase_diag*

These scripts:

- Run parameter sweeps over thermodynamic and architectural variables
- Execute parallel calculations to efficiently construct phase diagrams
- Produce serialized output used for downstream analysis and visualization

The directory also contains:

- Shell (.sh) job scripts used to run the workflows on the Stanford Sherlock cluster
- Templates that can be adapted to other HPC environments

Purpose:
Production pipeline for large parameter sweeps and data generation.

---

### figure_gen/

Figure reproduction workflow.

This folder contains a Jupyter notebook that:

- Loads output generated by the multipros_phase_diag workflows
- Processes and aggregates phase diagram data
- Reproduces all figures reported in the paper

Purpose:
End-to-end reproducibility of published figures.

---

## Workflow overview

Typical workflow:

1. Core theory is implemented in vertex_calc
2. Phase diagrams are generated using parallel scripts in data_gen_sherlock
3. Output data is passed to the notebook in figure_gen to reproduce figures
